## 💡 几点小建议（可进一步优化）

### 1. Phase 2 可以再拆细一点

当前 Phase 2（2-4 周）包含：
- TurnOrchestrator
- ToolRuntime
- Approval scope + grants + debounce + merge
- Subagent 审批上提
- Sandbox fallback 收紧

**建议拆成：**
```
Phase 2a：TurnOrchestrator + ToolRuntime（1-2 周）
Phase 2b：Approval scope + Sandbox fallback（1-2 周）
```

这样如果第一部分卡住，不影响第二部分。

---

### 2. ApprovalRequest 模型可以简化

当前设计：
```typescript
action: {
  type: "bash" | "file_read" | "file_write" | "network" | "mcp_tool" | "secret_access";
  command?: string;
  path?: string;
  domain?: string;
  toolName?: string;
};
```

**建议简化为：**
```typescript
action: {
  type: string;
  description: string;  // 人类可读描述
  fingerprint: string;  // 用于匹配 grant
  payload?: Record<string, unknown>;  // 可选扩展
};
```

**原因：** type 会持续增长，不如直接用 string + description。

---

### 3. "暂不做清单"可以更清晰

当前清单很好，但可以补充时间维度：

```typescript
// 第一版不做，但中期可以考虑：
SkillLoader 目录兼容增强
Memory 写入治理
Plugin 能力规范化

// 第一版不做，长期再看：
插件市场
多实例分布式部署
第三方插件隔离进程
```

